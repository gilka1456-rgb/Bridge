# 03 · 扫描鲁棒性:四个现实问题的对策

机主提出的扫描阶段问题,逐一给出架构级对策。核心思想只有一条:

> **把「对齐负担」从用户身上转移到关键点坐标系上。**
> 用户不可能四向姿势完全一致、不可能永远站直、不可能永远全身入镜 ——
> 但 33 关键点在任何姿势/视角/半身下都可用,一切归一化都锚定关键点即可。

---

## 问题 1:四向不一致(姿势在转身间变化)

挥手、呼吸、重心移动 —— 四个视角捕到的不是同一个「雕像」。剪影交集把不一致处全部切掉。

**三层防御:**

1. **采集期 · 姿势一致性门(T3)**
   - 每次捕获计算 `jointSignature`:8 个关节角(左右肩外展、肘弯、髋外展、膝弯);
   - 与已捕获视角比较,任一关节差 >25° → 该向不入库,语音+文字提示
     「保持与正面相同的姿势,重新转到 X 方向」;
   - 引导语明确要求:**双臂自然下垂或 A-pose,整个扫描期间不变**。
2. **重建期 · 部位可信度(T5 内置)**
   - 即使门放行,手臂仍是最不稳定部位 → 收缩包裹只信任躯干/骨盆/大腿的外壳数据,
     手臂/头永远走模板。姿势轻微不一致不再破坏完整性。
3. **数据期 · 首向为基准**
   - 第一个捕获向(通常正面)的 `jointSignature` 作为会话基准;
   - 模板最终摆姿态用**正面向关键点**,其余向只贡献宽度信息。

---

## 问题 2:非四向视角(趴着、躺着、斜靠)

被扫者不一定站着;镜头也不一定水平。方位角估计(`estimateBodyAzimuth`
基于双肩深度)在躺姿下直接失效。

**对策:身体直立坐标系(T4)**

1. 由关键点定义身体主轴:`up = normalize(neck - pelvis)`
   (neck = 双肩中点,pelvis = 双髋中点);
2. 计算主轴与图像竖直方向的夹角 `tilt`;
3. **掩码与关键点整体旋转 `-tilt`**,进入「身体直立」画布后,再走锚定归一化 ——
   躺着的人旋转后与站着的人数据形态完全一致;
4. 方位角改在身体坐标系里估计:绕身体主轴的旋转角(仍用双肩相对深度,
   但先把关键点变换进身体系);
5. `tilt > 60°`(接近水平)时在 UI 提示「检测到躺姿,已自动校正」,
   给用户确认感;`tilt` 本身不阻断采集。

**边界**:趴姿(面部朝下)时 MediaPipe 关键点可见度大幅下降,若
`可见关键点 < 20` 则拒收该帧并提示调整,这是模型能力边界,不强行建模。

---

## 问题 3:单人自拍难以全身入镜

手臂长度有限,前置镜头装不下全身 —— 这是采集方式问题,分三档解决,推荐做 a+b:

**a. 分段扫描 + 关键点缝合(推荐,纯算法)**
   - 允许「上半身一圈 + 下半身一圈」两段采集;
   - 每段用各自可见的锚点归一化(上段:骨盆+头顶;下段:骨盆+脚踝);
   - 两段共享骨盆锚点 → 在归一化画布上直接纵向拼接成全身掩码;
   - UI:扫描页加「全身 / 分段」模式切换,分段模式下引导语分两轮。

**b. 模板补全(T5 免费赠品)**
   - 只有上半身数据时,模板的腿部用标准体型渲染,标记
     `reconstruction.partial = "upper"`;
   - 用户至少能立刻得到完整人形(下半身个性化欠缺),
     后续可补扫下半身升级。

**c. 引导优化(零算法,纯 UX)**
   - 实时显示「入镜比例」:可见关键点跨度 / 33,低于阈值时语音提示
     「把手机放远一点/靠墙斜放,退后两步」;
   - 支持后置镜头 + 语音倒计时模式(手机靠墙,人退到 2.5m 外),
     现有语音引导体系直接复用。

---

## 问题 4:照片导入建模(测试路径转正)

本次测试用生成图模拟四向,暴露了一个真实需求:**用 2–4 张照片直接建像**
(自己的旧照片、他人帮拍的照片)。

**对策:照片导入作为正式采集 Provider(T8)**

1. 与活体扫描共用同一管线:照片 → MediaPipe 静态图模式(关键点+分割)→
   锚定归一化 → OrientationMask v3;
2. 方位角不再依赖转身流程,由「双肩深度差 + 用户手动指认」双保险:
   自动判向,同时 UI 允许拖拽纠正(正/背/左/右四个槽位);
3. 质量门与活体一致(mask 质量分 + 关节角一致性),不合格照片给出
   具体原因(「背景太乱/人太小/姿势与正面差异过大」);
4. 隐私:照片仅本地处理,提取掩码后原图不入库(与现有隐私政策一致)。

---

## 汇总:采集管线全景

```
输入源:活体扫描(整圈/分段) | 照片导入(2–4 张)
   │
   ▼
每视角:MediaPipe → 33 关键点 + 分割掩码
   │
   ├─ 可见关键点 <20 → 拒收,提示
   ├─ [T4] tilt 校正(躺姿→直立)
   ├─ [T3] jointSignature 一致性门(>25° → 重拍)
   ├─ [T1] 锚定归一化(骨盆+锚高)
   ├─ [T2] 双线性栅格化 + 平滑
   ▼
OrientationMask v3(锚点 + 关节签名 + 质量分)
   │
   ▼
[已有] SDF 软雕刻 → [T5] 模板混合 → [T6] 柔和灵体材质
```
