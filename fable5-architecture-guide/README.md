## 当前差距

默认路径仍使用 `template-body.ts`，新路径继续由版本开关保护，避免在真机验收前影响现有功能。
`ghost-body-v3=1` 当前生成一张连续水密人体：成人比例由单一常量表同时驱动 SDF 与 17 骨 rig，
胸、腰、骨盆使用连续截面场，肩部使用斜向肩带，头部使用颅骨/下颌复合体，手臂、手掌与腿部
使用连续宽度渐变；裆部、大腿和膝部截面均保持双腿分离。`ghost-render-v3=1` 启用共享透明渲染
内核，V5.2 与 V6.2 只在材质表现层分支。当前桌面自动门通过，最终游戏级视觉和 iPhone 真机门仍未完成。

## 改进主线（一张图）

```
采集（活体扫描 / 照片导入）
  │  每视角：分割掩码 + 33 关键点
  ▼
[T4] 躺姿/斜姿 → 关键点主轴旋转进「身体直立坐标系」
  ▼
[T1] 关键点锚定归一化（骨盆定位 + 关键点身高定尺，替代包围盒）
  │        └─ [T3] 姿势一致性门（各视角关节角差异过大 → 拒收并引导）
  ▼
[T2] 双线性栅格化（消横纹）→ OrientationMask v3（带锚点）
  ▼
SDF 软雕刻（已有）──► 躯干/腿部宽度场
  ▼
[T5] 模板人体混合：关键点摆姿态的低模人体 + 外壳收缩包裹
  │        └─ 保证任何输入下都是「完整分明的人形」
  ▼
[T6] 柔和灵体材质（淡彩低发光 + 软菲涅尔 + 洋葱壳 + 脚部消散，默认关扫描线）
```
