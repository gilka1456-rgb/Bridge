# 06 · Spectral V3 实施任务卡

执行顺序：`V0 → V1 → V2 → V3 → V4 → V5 → V6 → V7`。  
每卡独立 commit；单测、生产构建和该卡专属验收通过后才能进入下一卡。  
只推送 `gilka1456-rgb/ui-refactor`，不拉取、不修改 `main`。

## V0 · 契约、开关与现状 Golden 基线

**目标**：在任何几何变化前固定“现在长什么样”和新数据边界。

**内容**：

1. 新增 `GhostBodyModel`、`GhostRig`、`GhostLodMesh`、测量值和质量指标类型；
2. 固定 17 骨名称与顺序，加入契约单测；
3. 新增 `ghostBodyV3` / `ghostRenderV3`，默认关闭；
4. 建立确定性视觉基线页：固定人体、相机、旋转、时间、黑/白背景和旧奇幻/赛博参数；
5. 保存当前正/侧/背/三分之四视角截图与 manifest（commit、viewport、DPR、参数）；
6. README 与 `MAC_INTEGRATION.md` 只记录新契约，不改 iOS/main 实现。

**验收**：类型与 build 通过；契约测试能发现骨顺序或属性格式漂移；同一构建重复捕获的
轮廓边界一致；仓库中存在可直接比较的旧几何基线。

## V1 · 解剖隐式场与单档连续人体

**目标**：首次用一张连续水密人体替代管链视觉。

**内容**：

1. Worker 中建立标准 A-pose 17 骨和身体测量值；
2. 变半径圆角锥/椭球 SDF + 多项式 smooth-min；
3. visual-hull 场做 3×3×3 平滑、部位置信度和 ±4cm 限幅融合；
4. 1.8cm 场运行 MC；Taubin 仅平滑位置；中心差分生成场梯度法线；
5. 输出 regionId、chainT、canonicalCoord，先以静态标准姿态预览；
6. `ghostBodyV3` 开关控制新旧几何，失败自动回旧管链并报告原因。

**验收**：水密、单连通、无退化面；大腿截面双环、胸部单环；四视图截图中头颈肩胯
和四肢没有端盖接缝；与 V0 并排可见地降低几何人偶感。

## V2 · 程序化蒙皮与扫描姿势烘焙

**目标**：让连续标准人体可靠摆回扫描姿势。

**内容**：

1. 最近四骨距离权重、同链优先和腋下/胯部保护；
2. skin index/weight 量化为 `Uint8×4`，使用前重归一；
3. 实现 CPU `bakePose`，以捕获姿势生成静态 `BufferGeometry`；
4. reconstruction provider 缓存 `GhostBodyModel`，旧网格缓存版本隔离；
5. 照片导入、实时扫描和 partial 上半身都走新 provider。

**验收**：权重和误差 ≤1/128；无 NaN；极端举臂、屈肘、分腿姿势不破面；四张既有
测试图生成连续完整人体；缓存命中不重复重建。

## V3 · 共享渲染内核与透明深度正确性

**目标**：先解决结构可读与透明正确性，再添加风格特效。

**内容**：

1. 共用 shader chunks：蒙皮、位移、`structuralMask`、连续外观 alpha；
2. Pass 0/1 同步裁剪；Pass 2 additive 背壳；预留 GPU 粒子 pass；
3. 使用 canonicalCoord / regionId / chainT，禁止世界坐标随机噪声；
4. 显式 sRGB、NoToneMapping、预乘 alpha 与 AR additive 衰减；
5. 旧四风格先映射为新内核参数，不添加复杂粒子；
6. `ghostRenderV3` 独立控制并可回滚旧 shader。

**验收**：手臂不会透过躯干形成双影；结构缺口没有隐形深度壳；暗/亮背景轮廓可读；
确定性时间下重复截图稳定；构建与 shader 编译测试通过。

## V4 · 三档 LOD、SkinnedMesh 与质量控制器

**目标**：建立移动端可控的几何和渲染预算。

**内容**：

1. 同一融合场以 2.8cm / 4.2cm 再生成 LOD1/2，各自生成权重与属性；
2. `SkinnedMesh` 接管运行时姿势，保留 `bakePose` 测试/回滚路径；
3. 距离 LOD 与性能档位合并；
4. performance probe 输出 120 帧 P95、慢帧率、档位和 draw/triangle 统计；
5. 首先只记录降级建议，确认稳定后再启用自动切换。

**验收**：强制三档无崩溃和明显体型跳变；三角与 draw 落入预算；档位滞回不会在阈值
附近来回抖动；桌面压力测试通过。真机数据仍标记待验证。

## V5 · 奇幻灵体（红灵 / 白灵）

**目标**：完成第一种游戏级风格家族。

**内容**：

1. 双层标准坐标低频噪声、沿 chainT 流动、软肩能量曲线；
2. 柔和菲涅尔、近景背壳和 ≤0.5cm 确定性轮廓扰动；
3. 表面预采样出生点，GPU 驱动 300/120/0 三档粒子；
4. 红灵与白灵只换 palette/preset，不复制 shader；
5. 白底白灵增加冷灰结构暗部，黑底红灵控制高光溢出。

**验收**：两配色四视角与 10 秒序列；没有扫描线、果冻抖动、随机闪烁；主体在黑/白/
相机背景均完整可读；用户视觉确认达到目标方向。

## V6 · 赛博人物投影

**目标**：完成第二种游戏级风格家族。

**内容**：

1. 细扫描带、慢主扫描带、体空间块噪声；
2. 种子化相位事件：局部切片短时横移并复位；
3. structuralMask 驱动受限数据缺失，主表面保持绝大部分时间完整；
4. 体表青/品红镶边替代后处理色散；
5. additive 落地光盘；移除 wireframe；
6. 与奇幻共享几何、深度、粒子基础和质量档位。

**验收**：四视角与 10 秒序列；无火柴人/线框退化；缺块与深度一致；相位事件不造成
肢体永久断裂；黑底不过曝，亮背景仍可读；用户视觉确认达到目标方向。

## V7 · 端到端、压力与集成交付

**目标**：证明两风格在真实流程中可用，而不是只在测试人体上成立。

**内容**：

1. 四张既有生成图、partial 上半身、低质量输入、实时扫描数据全回归；
2. 单灵体/三灵体、近景/远景、旋转、透明相机合成压力场景；
3. 5 分钟性能采样与 10 分钟热衰减；
4. 缓存失效、fallback、取消重建和页面释放检查；
5. 更新 `MAC_INTEGRATION.md` 的二进制布局、骨顺序和风格参数镜像说明；
6. 全量测试、生产构建和最终截图/视频证据归档。

**最终门**：所有自动化证据通过；两种风格视觉获用户确认；iPhone 真机完成单/三灵体
30 FPS 底线与 P95 记录。真机证据缺失时可继续开发，但不得宣布整个目标完成。
