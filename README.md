# Bridge

Bridge 是一个“真人扫描 → 3D 虚像 → 场景展示”的产品原型。当前仓库同时包含：

- Web/3D 原型：在 Windows 和移动端浏览器中完成人体扫描、3D 建模、虚像预览、模拟放置与内容记录。
- iOS 原生 App：使用 SwiftUI、ARKit 与 RealityKit 实现真实空间中的放置和重定位。

Web 端的“看见”和“放置”是相机叠加与 3D 场景模拟，不是空间 AR；真实空间追踪属于 iOS 原生 App。

## Web/3D 当前能力

### 人体扫描

- 使用 MediaPipe Pose Landmarker 获取 33 点人体姿态。
- 使用 MediaPipe Image Segmenter 生成人体二值分割掩码。
- 优先使用 GPU，初始化失败时自动回退到 CPU。
- 根据肩部深度估计正面、右侧、背面、左侧四个朝向。
- 按人物垂直覆盖、边缘裁切和掩码面积计算采集质量；四个真实朝向全部有效后才能完成扫描。
- 支持实时覆盖率、转身提示、语音引导和退出保护。
- 每个朝向融合 5 帧稳定分割结果，并将人体等比例居中到统一的 `128 × 256` 坐标画布。
- 各朝向规范化掩码使用 base64 + RLE 保存到 `AvatarPose.orientations`，同时记录宽高比、帧数和质量。

### 3D 建模与渲染

- 将多朝向人体掩码转换为有符号距离场，在 `64 × 128 × 64` 体素空间执行带误差容忍的 Visual Hull 雕刻。
- 使用共享顶点的 Marching Cubes 生成网格，再通过 Taubin 平滑和法线重算获得连续表面。
- 共享解剖场使用连续头颈剖面构造颈部、下巴、下颌与颅顶，并以独立的脚跟—前掌纵横比例生成足部；体素边界偶发的一格级孤岛会被清除，而任何真正脱离的人体部位仍会触发质量失败。
- 雕刻、网格生成和平滑在 Web Worker 中执行；结果按源数据哈希缓存在 IndexedDB。
- 新扫描只有在完整人体网格通过验证后才能保存，不再静默回退为火柴人；旧数据使用兼容重建或连续圆润人形近似。
- 兼容灵体、幽灵、赛博、量子四种既有存档 ID；新建虚像只展示“奇幻灵体”和“赛博投影”两个家族，颜色由用户独立选择，渲染器自动派生暗部、轮廓色、粒子色和赛博辅色。
- 正式 Web 路径默认启用连续人体、V3 渲染、V5 奇幻灵体与 V6 赛博投影；奇幻和赛博效果会按存档风格自动分流，不会叠加。紧急回退可分别使用 `ghost-body-v3=0`、`ghost-render-v3=0`、`ghost-fantasy-v5=0`、`ghost-cyber-v6=0`。
- 虚像库支持旋转预览、删除和重复使用已保存扫描结果。

### 当前产品流程

Web 端包含五个主入口：

1. `看见`：相机画面与虚像叠加，支持全部、别人、自己三种筛选及快门拍摄。
2. `虚像`：扫描新虚像、查看虚像库和旋转预览。
3. `放置`：选择虚像、调整位置与方向、保存本地模拟放置。
4. `记录`：从“我的照片”选择图片发布本地记录，并支持点赞、评论与分享。
5. `我的`：管理照片、放置、个人资料、好友、聊天和设置。

当前 Web 原型没有后端：

- 结构化数据保存在 `localStorage`。
- 照片和头像媒体保存在 IndexedDB。
- 发布记录时会复制独立媒体资产，删除源照片不会同时删除帖子图片。
- 启动时会迁移旧数据并清理没有引用的媒体文件。
- 漂流模式会隐藏评论和社交入口，只保留点赞能力。

## 在 Windows 运行 Web 原型

要求：

- Node.js 20 或更新版本。
- 支持 WebGL、摄像头和 IndexedDB 的现代浏览器。
- 首次加载 MediaPipe WASM 和模型时需要网络连接。

在仓库根目录执行：

```powershell
npm.cmd --prefix core/web ci
npm.cmd --prefix core/web run dev
```

开发服务器默认地址为 `https://localhost:5173`，也会监听局域网地址。首次打开时浏览器可能提示本地开发证书风险。

摄像头 API 需要 HTTPS 或 localhost。手机测试时应打开开发服务器提供的 HTTPS 局域网地址，并在浏览器中允许摄像头权限。

如果当前 PowerShell 允许直接运行 `npm`，也可以使用：

```powershell
cd core/web
npm ci
npm run dev
```

## 测试与构建

在仓库根目录执行：

```powershell
npm.cmd --prefix core/web run test:run
npm.cmd --prefix core/web run build
```

提交 Web/3D 修改前，这两个命令都应通过。

`scripts/static_audit.sh` 和 Xcode 编译属于 macOS/Linux 或 CI 环境检查；Windows 端无法直接运行时不视为 Web/3D 阻塞。

## 当前 Web/3D 开发重点

当前沿 `fable5-architecture-guide/06-spectral-v3-task-cards.md` 实施 Spectral V3。下列条目保留架构演进记录；当前发布组合为共享人体 V20、共享渲染核心 V28、奇幻灵体 V5.29、赛博投影 V6.25、风格感知后处理 V1.1、双模式相机 V1。共享人体 V20 保留 V19 的胸腹骨盆 S 形中心线和四肢软组织偏移，并把肩关节横向位置从肩宽的 47% 收至 42%、上臂半径从肩宽的 14% 收至 11.5%，同时用连续的肩—斜方肌—颈部剖面替代骤缩平台，降低宽肩、短颈和几何分段感；共享渲染核心 V28 使用确定性的三角面面积采样并插值法线、区域和四骨权重，魂屑沿人体表面切线向上流动，赛博信号锁定在投影表面，所有附着特效服从同一人体深度，并在材质编译时只保留当前风格与当前 LOD 可见的表面 hook。奇幻 V5.29 将人体坐标中的灰烬、侵蚀和魂流组合为单一表面高度场，再从实际投影表面导出同坐标空间的切线法线，消除镜头转动时假凹凸高光游动或翻向的问题；共享表面细颗粒也已改为连续体噪声，不再随时间跨整数格闪烁。赛博 V6.25 保留投影源的断续环段，但用连续相位推进替代每两秒一次的整体跳格。任意用户颜色在接近纯黑时会保留最低可见能量，在接近纯白时会保留亮边余量，中间颜色仍使用原值；赛博的中性色保留本风格对比辅色。现在所有 Spectral Pass 统一在线性空间完成颜色计算，经过保留 72% 以下层次的软肩高亮压缩后转换为 sRGB，最后才执行预乘透明；红灵、白灵和赛博双色的亮部不会再直接剪成纯色。自动测试会用项目当前安装的 Three.js 片段表递归展开全部启用材质，检查片段缺失、递归引用、结构闭合及颜色转换顺序；这不能替代真机 GPU 截图，因此视觉验收页还会把人体、渲染核心、当前风格、相机和后处理的精确版本写入截图标识与页面就绪数据，避免把缓存或旧构建误认为最新效果。奇幻以烟魂体积明暗、边缘散射和贴肤魂流为主；赛博以 92% 自发光投影响应、窄扫描载波、锁定式表面信号和受控双色色散为主，只保留 8% 结构受光。奇幻主表面最低遮挡保持 94%，赛博保持 86%；两者都不依赖过度透明的人体：

共享外观场现为 V3：四向照片提取的隐私安全明暗与宽尺度浮雕会在最终索引网格上执行两轮双边连续化，只消除相近值形成的视角接缝，强衣褶与宽尺度起伏由差值阈值保护；该步骤不移动几何、不增加清晰纹理，也不进入逐帧渲染循环。

1. V0 已固定 17 骨契约、功能开关和旧几何四视图基线。
2. V1 已建立解剖隐式场、受限 Visual Hull 融合和连续水密标准 A-pose；当前正式默认路径已升级为 V20，近、中、远三级源场分别采用 1.45/2.5/3.7cm 采样，在手部提高轮廓分辨率的同时仍保持单一连续人体。
3. V2 已完成程序化四骨权重、链式姿势烘焙和版本隔离的 `GhostBodyModel` 本地缓存。
4. V3 已完成结构深度预写、预乘透明主表面、加色背壳、sRGB/NoToneMapping 和确定性视觉证据；默认启用并保留独立回退开关。
5. V4 已完成同源三级融合场、预算内重网格、GPU `SkinnedMesh` 姿势、CPU 回退及 120 帧质量控制；V20 的三级重网格比例为 1.12/1.34/1.40，三角形预算为 45,000/10,000/5,000。实时场景现默认执行带迟滞的自动切档：持续超过 30 FPS 压力线才逐级降档，恢复需要 5 秒稳定余量并受 3 秒冷却保护；固定时间视觉基线保持关闭，手动强制 LOD 始终优先。像素倍率也与同一档位联动：高档不超过 2×、中档不超过 1.5×、低档不超过 1×，手机性能结果会同时显示当前 DPR，避免只降几何却仍被高分辨率透明材质占满填充预算。
6. V5.19 奇幻灵体以《黑暗之魂 3》／《艾尔登法环》的灵体视觉语言为方向：完整、致密的外表负责人体结构，能量染色、贴附表面的魂流、灰烬斑驳、柔和轮廓光和稀疏逸散颗粒负责灵体感，不再依靠透视人体内部。主表面遮挡提高到接近实体，并把扫描明暗、缓慢魂蚀和灰烬层叠加在最外层；背景不会从人体中心穿透，灵体感集中在表面与轮廓。原先位于表面内侧且关闭深度测试的魂流层已移到外表并重新启用深度测试。容易强化几何分段感的水平亮带已改为沿身体表面纵向上升的扭曲魂流，并用低频灰烬壳、细颗粒和受限法线扰动打破平滑塑料感，轮廓仍保持完整稳定。V5.19 使用“深色实体内核—同色表面魂流—高亮边缘雾焰”三层材质，主体遮挡下限保持 94%，表面与外壳的火焰相位完全使用全身归一化坐标，经过胸、肩、手、腿时不会重新起算或暴露几何分区。新扫描会从四向照片提取去颜色、去曝光和模糊化后的基础明暗，以及只反映衣褶与宽尺度起伏的二级浮雕场；朝向采用高次权重，让最正对该表面的照片占主导，减少背面信息串入正面。两种场都只烘焙到共享人体顶点，不移动几何、不保存清晰照片纹理；V5.19 再将二级浮雕转换成实时表面法线，使衣褶真正参与受光而不再像画在光滑人偶上，并保持几何轮廓稳定。幻想外壳改为沿法线起伏、向上流动且受侵蚀噪声控制的稀疏雾焰，不再是均匀放大的一圈光。旧虚像自动使用中性值。每向压缩为 64×128、4 bit，四向合计约 20 KB。轮廓层沿局部表面法线做毫米级推出，120/48/0 分档魂屑与贴地魂雾继续保留。颜色不再绑定红灵／白灵预设，用户可选择任意主色，系统自动派生同色系暗部、亮边、雾焰和粒子色；白色与灰色会保持中性，不再被自动配色染出暖色暗部。
7. V6.15 赛博人物投影采用“稳定完整人体 + 外层传输信号”的架构。主人体不再因相位事件平移、缺块或切片，始终保持连续前表面和遮蔽下限，不显示人体内部；短时相位事件只作用于外层回声、边缘色散和空间信号。原先容易形成低模方格感的矩形数据块已由连续的宽频与细频噪声场替代，投影皮肤保持平滑；照片基础明暗与宽尺度浮雕会共同生成克制的投影表面法线，让衣褶和体表层次参与扫描光照，同时不改变人体几何或稳定轮廓。细扫描载波、地面投影源、96/40/0 分档空间信号符号、自下而上的重建波和克制的主色／辅色色散继续保留，且全部使用全身连续坐标，不会在身体区域边界重置。用户自选主色会自动派生对比辅色，投影源与人物保持一致。
8. 两种风格共享同一人体、骨骼、深度、LOD、质量架构和地面交互接口；奇幻分支提供贴地魂雾，赛博分支提供投影阵列，后续风格无需改动人体建模即可接入自己的场景交互。V10 共享人体把头部纵向比例、男性腰胯过渡和双腿轴线收敛到成人范围，并用横向形成掌宽、纵深持续收窄的各向异性手部体元替换旧版对称鼓包手套，使肩、腰、胯、手腕和手掌不再呈现低模假人比例；V11 在同一连续场内增加朝掌心侧张开的短拇指体元，并降低手端对照片外壳的裁切权重，因此举手轮廓不再只有一根等宽平板，融合照片外壳后仍保持单一水密人体。抬臂姿势使用全身统一的四骨骼线性混合，CPU 回退与 GPU 渲染共用相同矩阵和权重；V12 将权重扩散收敛在第四骨骼换槽前，使最坏肩腋接缝从 3.02× 降到 2.60×，并以 2.7× 自动回归上限阻止胸口到肘部的三角拉膜复发。V15 又在共享蒙皮层加入局部肩部体积修正：它以肩—肘轴为中心检测抬臂后丢失的横截面半径，只补回压缩部分，并在上臂中段前平滑退出；标准姿势和已经保持正常的截面不会膨胀。蒙皮 V16 将同一原则扩展到肘部和手腕，分别围绕肘—腕轴与腕—掌轴恢复弯曲时丢失的截面，避免前臂变成细杆、手掌变成扁片。CPU 烘焙与 GPU 实时顶点路径使用相同阈值和强度，奇幻、赛博及其粒子／信号附着层因此保持一致。目标风格的人物浮动已从旧版 ±4cm 收敛到 ±6mm，地面交互会反向抵消人物微动并始终锁定在地板高度，不再出现底座悬浮。四张全身测试图已全部通过识别（单图轮廓质量 91%–97%，重建质量 90%）。姿势以标准骨盆为根节点，照片取景偏移不会再平移或压低人物；局部缺脚的正面帧会自动让位给完整背面帧，不再破坏躯干和双腿。V17 将手端改为同一水密场内的短掌体、四个相互连接且长度有层次的指簇和独立拇指，并保留 22% 的四向轮廓证据；蒙皮再利用食指—小指方向校正掌面翻转，并把指端平滑锁定到末端手骨，举手时掌部不会被腕部混合压成尖片或扁片。V18 在最终索引网格上增加两轮带折角保护的法线连续化：只平滑相邻表面的受光方向，不移动顶点或轮廓，并在 CPU 姿势烘焙后执行同样步骤；奇幻的实体魂壳和赛博的投影高光因此不会再共同放大细小数值场块面。
9. V13 共享人体重新标定膝下轮廓：小腿在膝下收窄后形成腓肠肌外扩，脚部由脚踝、后跟、前掌和较扁的脚尖连续融合，不再以圆柱端头结束；手长略微收回，四指端保持接近掌宽的圆钝轮廓。脚部在照片中像素少且经常被地面裁掉，因此脚踝以下逐步降低 visual-hull 雕刻权重，由解剖场守住连接；肩盖与上臂固定使用同一组四骨骼权重，避免网格变化后第四骨骼换槽重新拉开腋下接缝。几何缓存升级到 `anatomical-sdf-v13-distal-hull-guard`，旧的三连通失败网格不会继续复用。自动测试另覆盖四个方向均裁掉脚底时仍保持单连通、水密和零退化面。V14 将胸、腰、胯控制点之间的逐段平滑改为单调三次 Hermite 连续轮廓，并增加连续肩带截面，去除每个控制点处的水平平台；躯干核心与左右肩只在三组稳定的四骨骼调色板之间切换，避免权重排序变化把胸肩表面切成可见拼块。共享人体 V15 把前臂截面从上臂的 74% 提高到 80%，同步恢复掌宽、掌厚、指端和拇指张开范围，并强化肘下的自然肌腹；最粗 LOD 的采样步长微调为 3.9cm，仍严格保持 4000 三角形预算。几何 V16 以单一纵向剖面替代头部叠加椭球，连续控制颈部、下巴、下颌、颧部与圆形颅顶，并缩小锁骨体元的垂直外扩，避免头颈再次粘成蛋形或肩部平台；足部改用区分垂直厚度与横向宽度的脚跟—前掌链，长度、宽度与厚度均锁定在成人比例。Marching Cubes 输出还会删除低于总三角形 0.5% 的单体素数值孤岛，若脱离部分超过该阈值则直接判定建模失败，不能用清理步骤掩盖断肢。
10. 渲染核心 V16 取消旧版对主体脚部的地面结构裁切，并按每个重建人体的真实最低点计算独立地面对齐；人物的毫米级悬浮仍保留，但最低相位不会让脚底沉入预览地板。投影盘和贴地魂雾使用反向局部偏移继续锁定同一世界地面。扫描预览另按真实人体包围盒自动计算相机中心与距离，高矮不同、举手或脚部前伸时都保持头顶到脚尖完整入镜；该构图现在也用于视觉基线和单人物档案，但不会改动发现页的摄像头叠加镜头。四张全身测试图已重新走完正式识别和重建流程：4/4 通过、质量 90%、耗时约 651ms，双风格均显示完整脚跟、前掌和脚尖。
11. 当前预览采用适合手机审查且能容纳举手姿势的全身构图。用户提供的四向样图已重新走完正式识别与重建：4/4 通过、质量 90%，奇幻与赛博均能识别掌部、拇指和圆钝的多指轮廓，人体保持完整遮挡而不显示内部。加入最终画面合成前的桌面浏览器高质量档曾实测：奇幻约 230 FPS、P95 4.4ms、201,132 渲染三角形／8 次绘制；赛博约 220.4 FPS、P95 8.2ms、160,944 渲染三角形／7 次绘制，两者慢帧均为 0%。这些数据只保留为旧链路参照，不代表后处理 V1.1 的当前性能；下一步需要重新进行最新版浏览器截图与 iPhone 真机验收。
12. 风格感知后处理 V1.1 在共享建模与材质架构之后增加统一的最终画面合成，但为两个家族保留独立响应：奇幻只对既有高亮能量增加较宽、较柔的雾焰辉光；赛博使用更高阈值、更低强度和更窄半径，保持投影边缘、扫描线与双色信号的硬朗感。该阶段不会降低人体主体遮挡，也不会制造内部透视。高档按设备 DPR 运行并请求最高 4× 离屏多重采样，中档把合成分辨率降到 72%、降低强度并请求 2× 采样，低档完全绕过；实际采样数受设备上限约束，若设备不足 2× 则直接保留原画布抗锯齿并关闭辉光。透明相机合成同样绕过，避免破坏摄像头画面的 alpha。性能统计现在包含全部合成绘制和实际离屏采样数，而不是只记录最后一个全屏 Pass；黑底视觉基线会启用并记录精确后处理版本，白底透明基线保留无后处理对照。
13. 双模式相机 V1 将人物审查与空间放置明确分开：扫描结果、单人物档案、放置前预览和视觉基线使用 34° 低畸变人物镜头，并根据当前人体宽高和纵深自动退远，降低近侧手臂、胸部和腿部相对远侧轮廓的透视放大；人物审查场景还会一次性计算姿势烘焙后的精确边界，让举手、弯肘和分腿进入取景与脚底对齐范围，但继续使用原 GPU 网格实时渲染。发现页摄像头叠加与多人空间场景保持 45° 场景镜头且不承担这次边界计算，避免用人物审查构图替代真实空间关系或增加批量开销。自动测试比较同一人体包围盒在两种镜头下的近远尺度比，并校验相机边界与 CPU 姿势烘焙结果一致，确保人物镜头实际减少纵深透视且不会裁掉姿势，而不只是改变常量名称。
14. 持续保持扫描、媒体生命周期、本地数据迁移和旧渲染回退测试完整。
15. V20 将肩部外轮廓约束纳入最终索引网格回归：肩部横向跨度保持在身高的 24%–30%，颈部跨度保持为肩部的 42%–65%，并逐层检查肩峰到颈根不会突然反向增宽。该约束直接验证融合、重网格后的真实表面，而不是只检查生成参数；头部下缘与同一颈根剖面相接，因此修正斜方肌坡度时仍保留下颌和颅骨比例。
16. 渲染核心 V24／奇幻 V5.27 修正程序化凹凸的坐标空间：不再把人体坐标的三路噪声直接当作相机坐标法线偏移，而是合成为标量表面高度，并通过屏幕导数在当前表面切平面求梯度。灰烬、魂蚀和细微起伏因此真正贴在蒙皮后的外表，转动镜头时不会产生与人体脱离的假高光；自动材质回归同时阻止旧的混合空间向量路径重新出现。
17. 渲染核心 V25／赛博 V6.23 消除两处时间量化：共享表面颗粒由移动整数格哈希改为人体坐标中的连续值噪声，赛博地面投影环由每两秒跳一次的离散相位改为连续旋转相位与软阈值。两者仍保留细颗粒和数字分段视觉，但不会再通过同步闪烁或整圈硬跳强化廉价特效感；自动回归禁止旧的时间取整表达式返回。
18. 渲染核心 V26 将共享主表面拆成编译期风格 hook：奇幻材质只编译烟魂体积噪声、灰烬和散射路径，赛博材质只编译扫描、传输哈希和投影发光路径，关闭增强效果的旧核心回退则两套都不编译。人体、蒙皮、深度预写、外观场、颜色输出和 LOD 接口仍完全共享，新增材质回归逐一验证 `1/0`、`0/1`、`0/0` 三种互斥开关，避免移动端为不可见风格承担片元计算与寄存器压力。
19. 共享人体 V20 的比例验收已从 rig 常量扩展到融合、重网格和平滑后的最终 LOD0 表面。当前标准测试体的最终高度为测量身高的 101.8%，胯点位于全身高度的 51.3%，肩点位于 80.8%，下巴至头顶为 14.1%，手腕至指端为 10.1%；发布门分别约束为 98%–105%、49%–54%、79%–83%、13%–15.5% 和 8.5%–11.5%。三级 LOD 仍需保持整体边界差异小于 6 cm，四向视觉外壳融合不得把解剖包络移动超过 4 cm，因此后续材质、融合或重网格调整不能再次把躯体、腿和手拉成非成人比例。
20. 渲染核心 V27／奇幻 V5.28／赛博 V6.24 为任意用户颜色增加显示亮度安全区。仅当输入接近全黑或全白时，主体亮度会收敛到 5.5%–90% 的线性范围，使深色仍有完整人体能量、浅色仍留有轮廓高光空间；范围内颜色逐值保持不变。暗部和亮边继续由同一色相派生，奇幻粒子跟随安全映射后的主体色；赛博灰、白、黑等中性色保留风格预设的对比辅色，不再因为无定义色相被意外派生成暖色。自动回归覆盖纯黑、纯白、灰色和高饱和中间色。
21. 渲染核心 V28／奇幻 V5.29／赛博 V6.25 将主表面细节与三级 LOD 一并编译为 `2/1/0`。近景保留完整灰烬微法线、魂流噪声、赛博闪点与双层时序传输；中景保留主体起伏和传输完整性，但移除只在近景有意义的最高频微纹；远景继续保留完整人体遮挡、奇幻低频烟魂和赛博扫描载波，同时不再计算会混叠的灰烬孔隙、微法线、闪点和时序哈希。外壳、粒子、地面交互与空间信号原有分档不变，材质与集成回归同时验证两种风格每档确实获得 `2/1/0`，而不是只改变 LOD 名称。
22. Web 真机性能页现提供四种可重复验收模式：1 人 5 秒快速检查、1 人 5 分钟正式测试、3 人 5 分钟多人测试，以及 3 人 10 分钟热衰减测试。默认压力风格已修正为完整赛博投影 V6，不再出现文字声称测试 V6、实际却只启用基础赛博材质的偏差；也可切换到奇幻灵体 V5。页面记录平均 FPS、最近 120 帧 P95、慢帧比例、运行期间出现的画质档、最高 LOD、LOD2 时间占比、最低 DPR、最终绘制次数和三角形；支持的浏览器还会记录峰值 JS 内存，iPhone Safari 未开放该接口时会明确标为不可用。热衰减除通用 30 FPS／40ms／5% 门槛外，还要求 LOD2 采样占比不超过 5%。这些模式只建立证据采集入口，不替代最新版画面截图和真实 iPhone 运行结果。
