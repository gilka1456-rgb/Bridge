# Web 灵体 iPhone 性能验收

本页只验证 Web 端共享人体与两套灵体材质在 iPhone Safari 中的实际表现，不涉及 Xcode、ARKit 或 iOS 原生工程。

## 打开方式

在可被 iPhone 访问的 Web 地址末尾加入以下参数。页面会自动开始测试，也可以在页面内切换模式和风格：

```text
?fps-test=1&fps-mode=quick&fps-style=cyber-v6
```

可用模式：

| 参数 | 内容 | 用途 |
| --- | --- | --- |
| `quick` | 1 人，5 秒 | 确认页面和正式渲染链路能运行 |
| `single` | 1 人，5 分钟 | 单灵体正式性能证据 |
| `triple` | 3 人，5 分钟 | 多灵体正式性能证据 |
| `thermal` | 3 人，10 分钟 | 持续负载与热衰减证据 |

可用风格：

- `cyber-v6`：赛博投影 V6，也是默认重载风格。
- `fantasy`：奇幻灵体 V5。

## 验收顺序

1. 关闭 iPhone 低电量模式，保持 Safari 页面在前台。
2. 先运行 `quick`，确认完整人体可见且页面正常计数。
3. 分别运行 `single`、`triple` 和 `thermal`，每次完成后截取包含结果数字的整页截图。
4. 奇幻与赛博至少各完成一次 `single`；较重的 `triple` 和 `thermal` 默认使用 `cyber-v6`。
5. 记录 iPhone 型号、iOS 版本、浏览器版本和测试 commit，避免把缓存或旧构建当作最新结果。

## 通过标准

- 平均帧率不低于 30 FPS。
- 最近 120 帧 P95 不高于 40ms。
- 最近 120 帧慢帧比例不高于 5%。
- 10 分钟热衰减测试中，LOD2 采样占比不高于 5%。
- 页面会同时显示运行期间的画质档、最高 LOD、最低 DPR、绘制次数和三角形。
- 峰值 JS 内存只在浏览器提供相应接口时显示；iPhone Safari 显示“未提供”不是测试失败。

最终完成 3D 建模目标仍需同时提供最新版四向画面和至少一次真实 iPhone 结果；桌面浏览器 FPS、旧截图和自动测试不能代替这两类证据。
