# Bridge 照片与空间隐私规则

本规则适用于「看见 → 我的照片 → 记录」链路。当前 Web 版本是本地原型；联网版和 iOS 正式版必须遵守同一语义。

## 数据边界

- `CapturedPhoto`（我的照片）默认仅保存在设备本地，不自动上传。
- 只有用户主动发布后，独立的帖子媒体资产才可上传；删除本地原图不影响帖子，删除帖子也不影响本地原图。
- 帖子不公开精确坐标、门牌、楼栋、房间号或原始定位元数据。公开地点由用户单独填写，默认降级为模糊地点。
- Web 合成画面仅用于交互原型；iOS 使用 `ARView.snapshot`，但隐私规则相同。

## 出镜与虚像

- 发布前提示用户确认画面中没有不愿出镜的路人、住址、车牌等敏感信息。
- 自己的虚像被删除或撤回时，已发布照片不自动删除，但权利人可申请下架。
- 他人的虚像仍按原放置可见性展示；“只看别人/只看自己”只影响本机取景，不改变所有权或公开权限。

## 联网版最低治理能力

上线前必须具备：

1. 对帖子、评论和虚像的举报入口及原因分类；
2. 屏蔽用户后停止展示其帖子、评论和虚像；
3. 权利人撤回、运营下架与申诉流程；
4. 服务端审核、来源校验和审计记录；
5. 删除账户后的媒体与位置数据清理时限；
6. 默认模糊位置，精确位置不得进入公开查询或日志。

单机原型不伪造“举报成功”等联网状态；相关按钮应在服务端能力接入时启用。
